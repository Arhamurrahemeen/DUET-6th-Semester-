The 5 classes — memorize this table

A 1 – 126 255.0.0.0 /8 824 2²⁴ − 2 = 16,777,214 Very large orgs

Exam tip: 127.x.x.x is loopback — not Class A usable. Max hosts = 2ⁿ − 2 always (subtract network address and broadcast address). Number of networks in Class A = 2⁷ − 2 = 126.

Four key addresses for any network block

Network address
All host bits = 0. First address in the block. Cannot be assigned to a host. Identifies the network itself.
Example: 192.168.1.0
Broadcast address
All host bits = 1. Last address in the block. Cannot be assigned to a host. Packets sent to this reach all hosts.
Example: 192.168.1.255
First host
Network address + 1. First usable address.
Example: 192.168.1.1
Last host
Broadcast address − 1. Last usable address.
Example: 192.168.1.254

Slash notation (CIDR prefix) — how to read it

172.16.0.0/20 means: 20 bits are the network prefix, 12 bits are the host part.

Subnet mask: first 20 bits = 1, remaining 12 bits = 0
Binary: 11111111.11111111.11110000.00000000
Decimal: 255.255.240.0

Quick conversion trick: the "interesting octet" is the one that is not 0 or 255. Here it is the 3rd octet: 20 − 16 = 4 bits set → 11110000 = 240.

Subnetting — the core process

Subnetting borrows bits from the host part to create multiple smaller networks (subnets) from one larger block.

Key formulas (memorize all four):

Number of subnets = 2ˢ   (s = borrowed subnet bits)
Hosts per subnet = 2ʰ − 2   (h = remaining host bits)
Block size (subnet increment) = 256 − interesting octet of mask
Total addresses per subnet = 2ʰ   (includes network + broadcast)

Step-by-step subnetting method

General procedure
1

Write the IP address and subnet mask in binary. Identify the slash prefix.

2

Find the "interesting octet" — the octet where the mask is neither 0 nor 255.

3

Block size = 256 − (interesting octet value of mask). This is the increment between subnets.

4

Subnets start at 0, block size, 2×block size, 3×block size… in the interesting octet.

5

For each subnet: Network = first address. Broadcast = next network − 1. Hosts = network+1 to broadcast−1.

6

Number of subnets = 2ˢ. Hosts per subnet = 2ʰ − 2. Verify your work.

Q4(a) — Classful network design

Exact question — Batch-22 Final Q4(a)

A famous communication service provider company needs to establish an internetwork comprised of several networks. Design the internetwork based upon networks and number of hosts in each network. Provide your assumption by selecting any single class of IP4 addressing. Provide the network address, masking values, slash notation, host addresses, and broadcast address of each network and defend the selection of that specific class for the interconnection of networks.

Model answer — Class C example design
1

Assumption: Selected Class C addressing. First octet range: 192–223. Default mask: 255.255.255.0 (/24). Reason: Class C supports up to 254 hosts per network — suitable for small-to-medium branch offices of a communication company.

2

Network 1: IP = 192.168.1.0/24
Network address: 192.168.1.0
Subnet mask: 255.255.255.0
Host range: 192.168.1.1 – 192.168.1.254
Broadcast: 192.168.1.255
Max hosts: 2⁸ − 2 = 254

3

Network 2: IP = 192.168.2.0/24
Network address: 192.168.2.0
Subnet mask: 255.255.255.0
Host range: 192.168.2.1 – 192.168.2.254
Broadcast: 192.168.2.255

4

Defense of Class C: Class C was chosen because each network serves a branch office with fewer than 254 hosts. Class A (16M hosts/net) and Class B (65K hosts/net) would waste vast IP space for small offices. Class C provides the right granularity and efficient address utilization.

Note: You can choose Class A or B — the examiner rewards the correct class properties and a logical defense. Marks = correct addresses (2) + correct defense (1) + correct mask/slash (2).

Q4(b) — Subnetting: the main numerical

Exact question — Batch-22 Final Q4(b)

A commercial bank in Karachi has requested to devise a plan for interconnecting its branches across the city. The network engineer has selected the IP address 172.16.0.0 with subnet mask 255.255.240.0/20 and 172.25.0.0 with subnet mask 255.255.248.0/21. Design the subnets for this organization by finding out the number of subnets, number of hosts, and range of IP addresses of the first five subnets.

Full solution — 172.16.0.0/20
1

Part 2: 172.25.0.0 / 255.255.248.0 (/21)

Full solution — 172.25.0.0/21
1

2

Mask binary: 11111111.11111111.11111000.00000000 = 255.255.248.0 ✓
Interesting octet = 3rd (248)

3

Subnets: 2⁵ = 32 subnets

4

Host bits: 32 − 21 = 11 bits → 2¹¹ − 2 = 2048 − 2 = 2046 hosts per subnet

5

Block size: 256 − 248 = 8 (increment in 3rd octet)

8 bits — last octet only
16 bits — Class B default

172.16.32.255 — only 4th octet matters
172.16.47.255 — subnet 3 starts at .32.0, block size=16, so ends at .47.255
172.16.48.255 — that is subnet 4's broadcast
172.16.31.255 — that is subnet 2's broadcast

16 — common mistake
