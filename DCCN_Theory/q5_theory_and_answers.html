Two routing algorithms — what the exam tests

Both algorithms solve the same problem: find the shortest path from a source to all other nodes. They differ in what information each router needs and how they compute it. Understanding this contrast is what C-6 taxonomy demands.

Dijkstra — Link State
Type: Global / centralized
Info needed: Complete network topology (all link costs)
How: Build a least-cost tree greedily — always pick the cheapest unvisited node next
Complexity: O(n²) comparisons
Protocol: OSPF uses this
Analogy: You have a full map — plan the entire route upfront
Bellman-Ford — Distance Vector
Type: Decentralized / distributed
Info needed: Only neighbor costs (no global map)
How: Exchange distance tables with neighbors, apply BF equation iteratively
Complexity: O(V × E)
Protocol: RIP uses this
Analogy: Ask people on the road how far each destination is from them

Dijkstra — exact exam procedure

Notation you must use in the exam

N = set of finalized nodes (least-cost path confirmed)
D(v) = current best known cost from source to v
p(v) = predecessor of v on the least-cost path
c(i,j) = direct link cost between i and j (∞ if no direct link)
Initialization: D(source)=0, D(all others)=∞. N={source}.
Each step: Pick w not in N with smallest D(w). Add w to N. Update all neighbors v of w: D(v) = min(D(v), D(w)+c(w,v))

Bellman-Ford — exact exam procedure

The Bellman-Ford equation

D_x(y) = min over all neighbors v { c(x,v) + D_v(y) }

Translation: The cost from node x to destination y = the minimum over all of x's neighbors v of (cost to reach v directly + v's known cost to y).

Each node maintains a distance table — rows = destinations, columns = cost via each neighbor. The minimum across each row = best known cost.

Q1: In Dijkstra's algorithm, when is a node added to set N?
When all its edges are explored
When it has the smallest D(v) among all nodes not yet in N — its shortest path is then confirmed
When it receives an update from a neighbor
At the end of each round, all nodes are added simultaneously
Q2: In the Batch-22 graph (A–B=3, A–C=1, C–D=2), what is the least-cost path from A to D and its cost?
A→B→D, cost 8 (3+5)
A→C→D, cost 3 (1+2)
A→D direct, cost ∞ (no direct link)
A→B→C→D, cost 11 (3+7+1)
Q3: What does D_x(y) = min{ c(x,v) + D_v(y) } mean in plain English?
The cost from x to y equals the sum of all link costs on the path
Node x finds the cheapest way to reach y by checking: go to each neighbor v directly, then use v's known best route to y — take the minimum
Node x broadcasts its full routing table to all nodes in the network
The distance from x to y decreases by 1 each round until it converges
Q4: In the W–X–Y–Z graph, what is W's shortest path to Z after convergence?
Cost 5 via Y (W→Y→Z = 3+3)
Cost 3 via X (W→X→Z = 1+2)
Cost 2 via X (W→X = 1, X→Z = 1)
Cost ∞ — W and Z have no path
Q5: What is the key difference between Dijkstra and Bellman-Ford in terms of information required?
Dijkstra needs neighbor costs only; Bellman-Ford needs the full topology
Dijkstra needs the complete network topology (link-state); Bellman-Ford only needs direct neighbor costs (distance-vector)
Both require the same information — they differ only in computation speed
Dijkstra uses hop count; Bellman-Ford uses actual link costs

Q5(a) — Model answer framework (5 marks)

What to write in the exam

1. State the algorithm type (0.5 mark): Dijkstra's algorithm is a link-state routing algorithm. Each router has a complete map of the network topology obtained via link-state flooding. It computes the least-cost path from one source to all destinations.


2. Define notation (0.5 mark): N = set of finalized nodes. D(v) = current least-cost estimate to v. p(v) = predecessor on least-cost path. c(i,j) = direct link cost.


3. Draw the step table (3 marks): Show all 5 steps. Each row: N set, D(v),p(v) for all non-source nodes. Highlight which node was added each step. Show the update calculation explicitly at least once.


4. State the least-cost tree (1 mark): List all shortest paths: A→C (1), A→B (3), A→D=A→C→D (3), A→E=A→B→E (4), A→F=A→C→D→F (9).

Q5(b) — Model answer framework (5 marks)

What to write in the exam

1. State the algorithm type (0.5 mark): Bellman-Ford distance-vector algorithm. Each node only knows its direct neighbor costs. Nodes share their distance tables with neighbors. Each applies the Bellman-Ford equation iteratively until convergence.


2. State the equation (0.5 mark): D_x(y) = min over all neighbors v { c(x,v) + D_v(y) }


3. Show Round 0 tables (1.5 marks): Draw the initial distance table for each node — only direct link costs known, others = ∞.


4. Show Round 1 update (1.5 marks): Show the Bellman-Ford calculation for at least one node explicitly. State which entries changed and why. Show the updated tables.


5. State convergence (1 mark): State which round the tables stabilized. Give the final shortest-path table. State that the algorithm terminates when no further updates occur.
