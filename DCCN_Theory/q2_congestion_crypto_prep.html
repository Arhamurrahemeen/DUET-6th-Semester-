Why congestion happens

Congestion = too many packets in the network, routers start dropping them. TCP detects congestion and controls how many unacknowledged bytes it injects into the network using cwnd (congestion window).

The three phases — know each trigger, action, and formula

Slow Start (SS)

Trigger: Connection starts, OR after a timeout event.
Rule: cwnd doubles every RTT (exponential growth). Starts at 1 MSS.
Formula: cwnd = cwnd × 2 each RTT
Ends when: cwnd reaches ssthresh → switch to Congestion Avoidance.
Memory hook: "Slow" is the name — the growth is actually exponential/fast.

Congestion Avoidance (CA)

Trigger: cwnd ≥ ssthresh.
Rule: cwnd increases by 1 MSS per RTT (linear/additive growth — AIMD).
Formula: cwnd = cwnd + 1 each RTT
Ends when: A congestion event is detected (timeout or 3 dup ACKs).

Fast Retransmit / Fast Recovery (FR)

Trigger: 3 duplicate ACKs received (mild congestion signal).
Action: Retransmit missing segment immediately (don't wait for timeout).
New ssthresh: ssthresh = cwnd / 2
New cwnd: cwnd = new ssthresh (jump to CA — no slow start!)
Why different from timeout: 3 dup ACKs means some packets still getting through — less severe than full timeout.

Two congestion events — the key difference

Timeout (severe)
Network is very congested. No ACKs arriving.

ssthresh = cwnd / 2
cwnd = 1 MSS (restart from scratch)
Phase: Back to Slow Start
3 Duplicate ACKs (mild)
Packet lost but later ones arrived. Partial connectivity.

ssthresh = cwnd / 2
cwnd = ssthresh (NOT back to 1)
Phase: Go directly to Congestion Avoidance

Modern ciphers — beyond substitution

Q2(b) can ask about modern ciphers (DES, asymmetric key) OR about performing a substitution cipher. Either way, understand both.

Traditional vs Modern

FeatureTraditionalModern
OrientationCharacter-orientedBit-oriented
SecurityWeak (simple patterns)Strong (complex permutations)
ExamplesAdditive, Vigenère, transpositionDES, AES, RSA
Key typesSingle simple keyLong keys (56-bit DES, 128/256-bit AES)

Modern block cipher components

P-boxes (Permutation)
Transposition units — rearrange bits without changing them. Straight P-box, compression P-box, expansion P-box.
S-boxes (Substitution)
Substitution units — replace bit patterns with other bit patterns using a lookup table. Core of DES security.
Operations used
XOR, Bit shifting, Swapping elements, Splitting, Combining. These make the cipher nonlinear and hard to reverse.
Stream vs Block
Stream cipher: Encrypts one symbol at a time.
Block cipher: Encrypts groups of m bits at a time. DES uses 64-bit blocks.

DES — Data Encryption Standard

Input: 64-bit plaintext → Output: 64-bit ciphertext
Key: 56-bit secret key (64-bit with parity)
Uses 16 rounds of P-box + S-box operations
Symmetric — same key for encrypt and decrypt
Weakness: 56-bit key is too short → brute-force crackable today → replaced by 3DES and AES

Asymmetric key cryptography (RSA concept)

1
Key generation
Each user generates a key pair: public key (shared openly) and private key (kept secret). Mathematically linked but computationally infeasible to derive one from the other.
2
Encryption
Sender uses recipient's public key to encrypt. C = E(public_key, P). Anyone can encrypt for you — only your private key decrypts.
3
Decryption
Recipient uses their private key. P = D(private_key, C). Only the private key holder can read the message.
4
Digital signatures (reverse use)
Sign with private key, verify with public key. Proves authenticity and non-repudiation. Hash the message → sign the hash → recipient verifies with public key.

Data integrity — message digest

A message digest (hash) is a fixed-size fingerprint of a message. Any change to the message completely changes the digest. Used to verify integrity — not confidentiality.

Process: Sender: message → hash function → digest. Sends message + digest. Receiver: rehashes message, compares digest. If different → message was tampered.

Self-test — congestion control

Q1: TCP is in Slow Start with cwnd=8 and ssthresh=16. A timeout occurs. What are the new cwnd and ssthresh values?
cwnd=8, ssthresh=8 — timeout halves both
cwnd=1, ssthresh=4 — timeout resets cwnd to 1, ssthresh=8/2=4
cwnd=4, ssthresh=4 — timeout halves cwnd to ssthresh
cwnd=1, ssthresh=16 — ssthresh unchanged on timeout
Q2: cwnd=12, ssthresh=8. Three duplicate ACKs arrive. What happens?
cwnd=1, ssthresh=6 — 3 dup ACKs treated like timeout
cwnd=12, ssthresh=6 — cwnd stays, ssthresh halved
cwnd=6, ssthresh=6 — ssthresh=12/2=6, cwnd=ssthresh, enter CA directly
cwnd=8, ssthresh=8 — revert to previous ssthresh
Q3: What is the key difference between Slow Start and Congestion Avoidance?
Slow Start uses linear growth; CA uses exponential growth
Slow Start doubles cwnd each RTT; CA increases by 1 MSS per RTT
Slow Start only runs once per connection; CA runs repeatedly
Both grow at the same rate — they differ only in the ssthresh threshold
Q4: Which is more severe — a timeout or 3 duplicate ACKs? Why?
Timeout is more severe — no ACKs means total congestion; 3 dup ACKs means some packets still arriving
3 dup ACKs are more severe — they indicate multiple lost packets
Both are equally severe — they trigger the same response
Timeout is less severe — the sender just waits and retries

Q2(a) — Model answer (5 marks)

Exam answer framework

Para 1 — What is congestion control: Congestion occurs when too many datagrams enter the network, causing routers to drop packets. TCP uses congestion control to regulate the rate of data injection using a congestion window (cwnd). The effective window = min(cwnd, receiver window).


Para 2 — Slow Start: TCP begins with cwnd=1 MSS. cwnd doubles every RTT (exponential). Continues until cwnd reaches ssthresh, then switches to Congestion Avoidance.


Para 3 — Congestion Avoidance: cwnd increases by 1 MSS per RTT (linear/additive). This is the AIMD phase — Additive Increase, Multiplicative Decrease.


Para 4 — Timeout event: Severe congestion signal. ssthresh = cwnd/2. cwnd reset to 1. Return to Slow Start. This is the multiplicative decrease.


Para 5 — 3 Dup ACKs / Fast Retransmit: Mild congestion. Missing segment retransmitted immediately. ssthresh = cwnd/2. cwnd = ssthresh. Jump directly to Congestion Avoidance — not Slow Start. This avoids the performance penalty of restarting from 1.


For the graph: Label each phase region. Mark timeout and 3-dup-ACK points with arrows. Show ssthresh as a dashed horizontal line. State the cwnd and ssthresh values at each event.

Q2(b) — Model answer: cipher (5 marks)

Exam answer framework

If asked to perform a cipher on a word/name:

  • Write the formula: C = (P + k) mod 26 for encryption; P = (C − k + 26) mod 26 for decryption
  • Draw the full working table: plain letter → numeric value → + key → mod 26 → ciphertext value → cipher letter
  • Show both encryption AND decryption to verify
  • State the cipher type: monoalphabetic substitution (additive/Caesar cipher)

If asked to explain modern ciphers:

  • Distinguish traditional (character-based) vs modern (bit-based)
  • Explain stream cipher (one symbol at a time) vs block cipher (m bits at a time)
  • Describe DES: 64-bit input, 56-bit key, 16 rounds of P-box + S-box operations
  • Explain asymmetric key: two keys (public + private), encrypt with public, decrypt with private, used in digital signatures
