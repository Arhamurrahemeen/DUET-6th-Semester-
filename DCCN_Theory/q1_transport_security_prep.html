Transport Layer — core concepts

What it does
Provides logical (process-to-process) communication between applications on different hosts. Works above Network Layer (host-to-host). Uses port numbers to identify processes.
Addressing at each layer
Data Link → MAC address (node-to-node)
Network → IP address (host-to-host)
Transport → Port number (process-to-process)

Socket address — the key concept

A socket address = IP address + port number. It uniquely identifies a process on a host.

Transport layer needs a pair: client socket address + server socket address.

Example: Client 192.168.1.5:50123 connects to Server 142.250.1.1:80 (HTTP)

Three-way handshake — step by step

Step 1 — SYN: Client sends segment with SYN=1, seq=x. Client enters SYN SENT state.
Step 2 — SYN+ACK: Server replies SYN=1, ACK=1, seq=y, ack=x+1. Server enters SYN RECEIVED.
Step 3 — ACK: Client sends ACK=1, ack=y+1. Both sides enter ESTABLISHED state. Data transfer begins.

Client-server vs Peer-to-peer

Client-Server
Dedicated server waits on a well-known port. Clients initiate connections. Server is always on, has fixed IP. Examples: HTTP (web), FTP, Email (SMTP/IMAP).

Socket pair: Client socket + Server socket (well-known port like 80).
Peer-to-Peer
Each peer acts as both client and server. No dedicated server. Peers connect directly using TCP/IP sockets. Examples: BitTorrent, Skype (historically), blockchain nodes.

More scalable but harder to manage.

Security goals — CIA + A (memorize the acronym)

Confidentiality
Data hidden from unauthorized users. Broken by snooping and traffic analysis.
Integrity
Data protected from unauthorized modification. Broken by modification, masquerading, replaying, repudiation.
Availability
Services accessible to authorized users. Broken by Denial of Service (DoS) attacks.
Authenticity
Verify the identity of the communicating entity. Ensured via digital signatures.

Cryptography — symmetric vs asymmetric

Symmetric (same key)
Same key used for encryption AND decryption. Fast. Key must be shared securely (face-to-face or trusted channel).

C = Ek(P)    P = Dk(C)

Types: Traditional (substitution, transposition) and Modern (DES, AES).
Asymmetric (public/private)
Two keys: public key (shared openly) + private key (kept secret). Encrypt with public key, decrypt with private key. Used in digital signatures — sign with private, verify with public.

Traditional ciphers — substitution

Additive (Caesar/shift) cipher: Each letter shifted by key k.
Formula: C = (P + k) mod 26    Decrypt: P = (C − k) mod 26

Monoalphabetic: One plaintext letter always maps to same ciphertext letter.
Polyalphabetic: Same plaintext letter maps to different ciphertext depending on position (harder to crack).

Provides source authentication + integrity + confidentiality. Encrypts payload.

SSL/TLS and Firewalls

SSL/TLS — Transport Layer security
Provides security between application and transport layers. HTTP uses SSL → HTTPS. Creates cryptographic session keys via handshake.
Firewalls — two types
Packet-filter: Filters at Network/Transport layer (IP, port). Simple, fast.
Proxy firewall: Filters at Application layer. Understands application protocols — more intelligent.

Q1(a) — Model answer structure (5 marks)

Exam answer framework — Q1(a)

Para 1 — Role of Transport Layer: The transport layer provides process-to-process (end-to-end) delivery using port numbers. It sits above the network layer (which provides host-to-host delivery) and receives data from application layer processes. It performs: segmentation, end-to-end error control, flow control, multiplexing/demultiplexing.


Para 2 — TCP/IP Socket: A socket address is the combination of IP address and port number. Transport layer needs a pair — client socket and server socket — to establish communication. TCP uses this socket pair to create a virtual circuit (connection-oriented).


Para 3 — Client-Server: In this model, the server runs permanently on a well-known port. Clients initiate connections. Example: HTTP client connects to port 80. The server socket address is fixed; the client uses an ephemeral port.


Para 4 — Peer-to-Peer: Each peer acts as both client and server simultaneously. There is no dedicated server; peers discover each other and connect directly using TCP/IP sockets. Examples: BitTorrent file sharing, distributed blockchain nodes.

Q1(b) — Model answer structure (5 marks)

Exam answer framework — Q1(b): Attacks + Cipher

Part i — Security Goals (1 mark): CIA + Authenticity. Define each briefly.


Part ii — Important attacks (2 marks): List and explain 2–3: snooping (breaks confidentiality by intercepting plain data), DoS (breaks availability by flooding server), masquerading (breaks integrity/auth by impersonating), replaying (breaks integrity by re-sending old messages).


Part iii — Substitution cipher (2 marks): Show the full working table. Formula: C = (P + k) mod 26. Do encrypt and decrypt both. Show all steps — the marks are for the working, not just the answer.
